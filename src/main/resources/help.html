PGP GUI

PGP GUI is a program wrapping the most fundamental PGP functionality provided via the Bouncy Castle cryptography library in a graphical interface. Features include creating, listing, deleting and importing keys, as well as encrypting and decrypting messages and files using asymmetric PGP keys or a shared static password.

The program can read and use the key ring files managed by GnuPG. Having GnuPG installed is however optional — PGP GUI maintains its own public and secret key ring files and can operate fully without GnuPG.

List Keys

Lists either public or secret keys found in the GnuPG and/or PGP GUI key ring files. The Key Ring column indicates which key ring file the key belongs to (GnuPG or PGP GUI).

Double-clicking a key opens a menu to expand (view details) or delete the key.

Create Keys

Creates an asymmetric key pair including an encryption subkey. The supported algorithms are RSA and DSA/ElGamal. Keys can be saved to the GnuPG key ring files (if GnuPG is installed) or to the PGP GUI own key ring files.

Import Key

Imports an ASCII-armored public or secret key into either the GnuPG key ring files (if available) or the PGP GUI key ring files.

Encrypt Message

The Encrypt Message window provides two encryption methods, selectable via tabs:

  • PGP Encryption — encrypts the message using a recipient's PGP public key and a chosen symmetric algorithm (AES, CAST5, Blowfish, Camellia-256, IDEA or Twofish). Optionally the message can be signed with your own secret key.
  • Static Password Encryption — encrypts the message using a shared password, requiring no PGP keys. See Static Password Encryption below.

Right-clicking the text area opens a context menu with Copy to Clipboard and Paste from Clipboard options.

Decrypt Message

Decrypts an encrypted message pasted into the text area. The program automatically detects the message type from its header:

  • PGP-encrypted messages (-----BEGIN PGP MESSAGE-----) are decrypted using the matching secret key found in the GnuPG or PGP GUI key ring files.
  • Static-password-encrypted messages (-----BEGIN STATIC ENCRYPTED MESSAGE-----) prompt for the password used during encryption.

Right-clicking the text area opens a context menu with Copy to Clipboard and Paste from Clipboard options.

Encrypt File

Encrypts a file using a recipient's PGP public key. The output can optionally be ASCII-armored.

Decrypt File

Decrypts a PGP-encrypted file using the matching secret key found in the GnuPG or PGP GUI key ring files.

Static Password Encryption

Static Password Encryption allows messages to be encrypted and decrypted using only a shared password, with no PGP keys required. This is useful for communicating with parties who do not use PGP.

The encrypted output is ASCII-armored for safe copy-paste: 

-----BEGIN STATIC ENCRYPTED MESSAGE-----
... Base64 encoded ciphertext ...
-----END STATIC ENCRYPTED MESSAGE-----

The technical parameters used are:

  • Cipher: AES-256 in CBC mode with PKCS5 padding
  • Key derivation: PBKDF2-HMAC-SHA256
  • Iterations: 65,431
  • Salt: MANDELBROTIAN
  • IV: Randomly generated 16-byte initialisation vector, unique per encryption
The same password must be used for both encryption and decryption. There is no way to recover a message if the password is lost.

Settings

The Settings dialog allows the file paths of the GnuPG and PGP GUI key ring files to be configured. It also controls how long a secret key password is cached after first use, avoiding repeated password prompts within a session.
